Tectionário

  • Back-End(Parte traseira) - Programadores Back-End trabalham com a lógica da aplicação, com o armazenamento e a segurança de dados.
  • Bit (Pedaço) - É a menor unidade de medida de dados que pode ser armazenada ou transmitida no universo computacional. Um bit tem um único valor, sendo zero ou um, ou seja, verdadeiro ou falso.
  • Box Collider (Caixa de Colisão): É uma caixa de colisões padrão, onde trabalhando juntamente do corpo rígido, permitem que os objetos sejam controlados pela física.
  • Branch (Galho) - São as ramificações de um repositório. Cada branch tem a mesma origem, mas têm suas próprias configurações.
  • Bug (Erro) - Bug é um erro ou falha que ocorre e faz com que os programas não se comportem exatamente da maneira esperada. Também é muito conhecido como “dar pau”.
  • Build (Construir) - Assim como o significado do seu nome, "constrói" a cena para ser apresentada e testada como uma versão executável do jogo, possibilitando que os desenvolvedores possam testar as diferenças aplicadas na programação.
  • ChatBot (Robô do Chat) - São programas de computador equipados com inteligência artificial capazes de entender a linguagem humana, tanto escrita como falada, e responder de forma humanizada. São como aqueles atendentes de empresas telefônicas.
  • Commit (Entregar): É quando é feita alguma alteração individual por um usuário em algum arquivo e ele é entregue ao principal.
  • Developer (Desenvolvedor) - Geralmente abreviado para Dev, é um termo muito usado na comunidade de programação para designar alguém que estuda ou trabalha com desenvolvimento de software.
  • Engine (Motor) - Plataforma usada para produzir o jogo e suas mecânicas, com mecanismos integrados como a gravidade.
  • Full Stack (Pilha Completa) - Programador full-stack é aquele que atua em várias partes do projeto, ele entende tanto de back-end como de front-end e usa várias tecnologias.
  • FPS/Frame Per Second (Quadros por segundo): É uma unidade de medida audiovisual, qualquer ação ou movimento que vemos em jogos e filmes é feita por esses quadros, onde quanto mais quadros em apenas um segundo, mais fluída é a imagem.
  • Framework (Estrutura) - São como templates prontos que podem ser usados em diferentes projetos (back e front-end, mobile etc), reduzindo o tempo gasto para produzir uma aplicação.
  • Front-End (Parte dianteira) - É a parte da web que é apresentada, aquela com a qual interagimos. Programadores Front-End se preocupam com a experiência do usuário (UX), além da interface do produto (UI). Instantiate (Instanciar) - Faz uma "cópia" de um objeto e o posiciona o clone na posição informada.
  • Keyboard (Teclado) - serve como um dispositivo para o envio de dados pelo usuário, que serão interpretados pelo computador e assim o programador pode usar todas as teclas como quiser (um exemplo são as setas, que normalmente usamos para mover o personagem).
  • Merge (Mesclar) - Assim como a tradução literal, merge é unir as funcionalidades que antes estavam sendo desenvolvidas separadamente.
  • Prefab (Pré-fabricados) - Um objeto com configurações conservadas para usar em vários momentos do jogo, como por exemplo um inimigo genérico que precisa aparecer diversas vezes no meio da produção.
  • Render (Renderizar) - Obter um produto final após um processamento digital (áudio, imagem, etc.). Melhora a visibilidade e a qualidade do projeto que é desenvolvido, geralmente, em plataformas 2D ou 3D.
  • Rigidbody (Corpo Rígido): São componentes que permitem o objeto interagir com a física em tempo real, como a gravidade, colisões e etc.
  • Shader (Sombreador) - Combinação de texturas para gerar um objeto com o aspecto semelhante à algo real.
  • Version (Versão) - O controle de versões é muito importante durante um projeto, principalmente se for em equipe, permitindo que as atualizações sejam feitas de forma organizada. Além do mais, é possível dessa forma ver mais facilmente a evolução do jogo.
  • Void (Vazio): É um método de retorno usado para assinar um método que não retorna valor nenhum.
  • While (Enquanto) - Repete os comandos (que o programador definiu) quantas vezes foram necessárias até uma condição ser atingida.
